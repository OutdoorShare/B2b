We have a critical auth/UI issue: login is not working anywhere, and it appears the form is not even attempting to submit.

Current behavior:

* user enters login info
* clicks submit/sign in
* nothing happens
* no visible loading
* no redirect
* no error
* it appears the login action is not firing at all

Please treat this as a full login submit pipeline failure, not just a credentials issue.

Goal:
When a user clicks login, the app must clearly do one of these:

* start loading and submit auth request
* show validation error
* show auth error
* redirect on success

It should never appear dead.

Please inspect the login form and confirm:

* no parent element is blocking the event
* no disabled/loading condition is stuck permanently
* no JavaScript error is preventing submit
* preventDefault is not swallowing the event without calling the login action

Add temporary logs for:

* login form mounted
* submit clicked
* onSubmit fired
* validation passed/failed
* auth request started
* auth request finished
* auth request error

Right now it feels like the click does nothing, so I need to know the exact step where it dies.
