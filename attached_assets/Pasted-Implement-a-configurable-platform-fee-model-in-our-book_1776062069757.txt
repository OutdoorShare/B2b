Implement a configurable platform-fee model in our booking flow with 3 modes:

1. Pass fees to customer
2. Absorb fees
3. Split fees

Goal:
Operators must be able to choose how platform fees are handled per listing or product, and all pricing must calculate correctly in the storefront, checkout, booking summary, admin dashboard, and payouts.

Core requirements

Add a new fee configuration object to each bookable listing/product:

feeSettings = {
mode: "pass_to_customer" | "absorb" | "split",
platformFeeType: "percentage",
platformFeePercent: number,
splitCustomerPercent?: number,
splitOperatorPercent?: number
}

Assumptions:

* Start with percentage-based platform fees only
* platformFeePercent is the total platform fee rate
* For split mode:

  * splitCustomerPercent + splitOperatorPercent must equal 100
  * Example: platformFeePercent = 10, splitCustomerPercent = 40, splitOperatorPercent = 60
  * Customer pays 4% of subtotal
  * Operator absorbs 6% of subtotal

Behavior by mode

1. pass_to_customer

* Customer pays subtotal + customer-facing platform fee
* Operator receives full subtotal, minus payment processor fees if those are handled separately
* Platform revenue = full platform fee

Formula:

* subtotal = base booking amount before platform fee
* platformFee = subtotal * platformFeePercent / 100
* customerTotal = subtotal + platformFee
* operatorGrossRevenue = subtotal
* platformRevenue = platformFee

2. absorb

* Customer pays subtotal only
* Operator payout is reduced by the platform fee
* Platform revenue comes out of operator earnings

Formula:

* subtotal = base booking amount
* platformFee = subtotal * platformFeePercent / 100
* customerTotal = subtotal
* operatorGrossRevenue = subtotal - platformFee
* platformRevenue = platformFee

3. split

* Part of fee is charged to customer, part is absorbed by operator
* Total platform revenue must still equal subtotal * platformFeePercent / 100

Formula:

* subtotal = base booking amount
* totalPlatformFee = subtotal * platformFeePercent / 100
* customerFee = totalPlatformFee * splitCustomerPercent / 100
* operatorFee = totalPlatformFee * splitOperatorPercent / 100
* customerTotal = subtotal + customerFee
* operatorGrossRevenue = subtotal - operatorFee
* platformRevenue = customerFee + operatorFee

Validation rules

* platformFeePercent must be >= 0
* split mode requires splitCustomerPercent and splitOperatorPercent
* splitCustomerPercent + splitOperatorPercent must equal 100
* No negative totals or payouts
* Round currency safely to 2 decimals using a single shared money utility
* Do not duplicate fee math across components; use one central pricing calculator

Need a central pricing calculator

Create a shared function like:

calculateBookingPricing({
subtotal,
feeMode,
platformFeePercent,
splitCustomerPercent,
splitOperatorPercent
}) => {
subtotal,
totalPlatformFee,
customerFee,
operatorFee,
customerTotal,
operatorPayout,
platformRevenue
}

Important:

* This function must be the single source of truth
* Use it everywhere:

  * product page
  * checkout
  * booking confirmation
  * admin panel
  * payout reporting
  * invoices/receipts

Pseudo logic:

function roundMoney(value) {
return Math.round((value + Number.EPSILON) * 100) / 100;
}

function calculateBookingPricing(input) {
const subtotal = roundMoney(input.subtotal);
const totalPlatformFee = roundMoney(
subtotal * input.platformFeePercent / 100
);

let customerFee = 0;
let operatorFee = 0;

if (input.feeMode === "pass_to_customer") {
customerFee = totalPlatformFee;
operatorFee = 0;
} else if (input.feeMode === "absorb") {
customerFee = 0;
operatorFee = totalPlatformFee;
} else if (input.feeMode === "split") {
customerFee = roundMoney(
totalPlatformFee * input.splitCustomerPercent / 100
);
operatorFee = roundMoney(totalPlatformFee - customerFee);
} else {
throw new Error("Invalid fee mode");
}

const customerTotal = roundMoney(subtotal + customerFee);
const operatorPayout = roundMoney(subtotal - operatorFee);
const platformRevenue = roundMoney(customerFee + operatorFee);

return {
subtotal,
totalPlatformFee,
customerFee,
operatorFee,
customerTotal,
operatorPayout,
platformRevenue
};
}
